
# CRM-система управления проектами

Веб-приложение для управления проектами, сотрудниками и встречами. Замена Google Sheets на структурированную CRM-систему.

## 📋 О проекте

Система позволяет:
- Управлять проектами (создание, редактирование, удаление, фильтрация)
- Вести базу сотрудников команды
- Планировать встречи и созвоны по проектам
- Фильтровать проекты по статусу, приоритету и ответственному
- Экспортировать и импортировать данные в формате CSV

## 🛠 Технологии

### Бэкенд
- **Node.js** — среда выполнения JavaScript
- **Express** — веб-фреймворк
- **Prisma** — ORM для работы с базой данных
- **PostgreSQL** — реляционная база данных
- **JWT** — аутентификация и авторизация

### Фронтенд
- **TypeScript** — типизированный JavaScript
- **HTML5/CSS3** — разметка и стили
- **Vanilla JS** — нативный JavaScript без фреймворков

### Инфраструктура
- **Docker** — контейнеризация
- **GitHub Actions** — CI/CD

## 🚀 Установка и запуск

### Требования
- Node.js 18+
- PostgreSQL 14+
- npm или yarn
- Docker (опционально)

### Локальный запуск

#### 1. Клонировать репозиторий
```bash
git clone https://github.com/pozikt/crm-backend.git
cd crm-backend
```

#### 2. Установить зависимости
```bash
npm install
```

#### 3. Настроить переменные окружения
```bash
cp .env.example .env
# Отредактировать .env, указав пароль PostgreSQL
```

#### 4. Создать базу данных и применить миграции
```bash
# Убедитесь, что PostgreSQL запущен
sudo service postgresql start  # Linux
# или
docker run --name crm-postgres -e POSTGRES_PASSWORD=ledenci -e POSTGRES_DB=crm_db -p 5432:5432 -d postgres:14

# Применить миграции
npx prisma migrate dev

# Заполнить тестовыми данными
node prisma/seed.js
```

#### 5. Запустить сервер
```bash
npm run dev
```

Сервер запустится на `http://localhost:3000`

#### 6. Запустить фронтенд (в отдельном терминале)
```bash
cd frontend
npx serve . -p 5173
```

Открыть в браузере: `http://localhost:5173`

### Запуск через Docker

```bash
docker-compose up -d
```

После запуска:
- Фронтенд: http://localhost:5173
- Бэкенд API: http://localhost:3000/api
- Документация: http://localhost:3000/api-docs

## 📡 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/projects` | Получить список проектов (с фильтрацией) |
| GET | `/api/projects/:id` | Получить проект по ID |
| POST | `/api/projects` | Создать новый проект |
| PUT | `/api/projects/:id` | Обновить проект |
| DELETE | `/api/projects/:id` | Удалить проект |
| GET | `/api/employees` | Получить список сотрудников |
| POST | `/api/employees` | Создать сотрудника |
| GET | `/api/meetings` | Получить список встреч |
| POST | `/api/meetings` | Создать встречу |
| GET | `/api/statuses` | Получить список статусов |
| GET | `/api/priorities` | Получить список приоритетов |
| GET | `/api/projects/export-csv` | Экспорт проектов в CSV |
| POST | `/api/projects/import-csv` | Импорт проектов из CSV |

### Параметры фильтрации

```
GET /api/projects?status=В%20работе
GET /api/projects?priority=Высокий
GET /api/projects?responsible=Иванов
GET /api/projects?search=CRM
```

## 📊 База данных

### Основные сущности

| Таблица | Описание |
|---------|----------|
| `projects` | Проекты |
| `employees` | Сотрудники |
| `meetings` | Встречи/созвоны |
| `meeting_participants` | Участники встреч |
| `project_statuses` | Статусы проектов |
| `project_priorities` | Приоритеты |
| `users` | Пользователи (аутентификация) |
